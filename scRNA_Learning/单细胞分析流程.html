单细胞数据的读取

几种常见的单细胞数据类型读取方法

  • 读取标准的10X数据

最常见和标准的数据类型，直接用Seurat包的read10X()函数读取：需要1个参数data.dir,即10X数据所在的文件夹，文件夹下的三个文件名必须为“barcodes.tsv.gz”，“features.tsv.gz”和“matrix.mtx.gz”，不需要解压缩，文件名也不能有其他前缀，否则会读取失败，提示找不到文件。

counts <- Read10X(data.dir = "3个文件所在路径") #读取稀疏矩阵
#创建seurat对象，其中的“样本编号”也就是构建对象后的orig.ident,即样本来源，
seurat_obj <- CreateSeuratObject(counts = counts,
                                 project = "样本编号",
                                 min.features = 200,  #至少200个基因表达
                                 min.cells = 3)   #至少3个细胞内有表达的基因

  • 读取H5数据

批量数据的读取

常见的单细胞数据基本上就是以上这些了，然而，单细胞数据往往不是单独的一个样本，而是数个或十数个样本，如果单个数据读入、质控再merge的话将会非常不便且容易出错，此时需要根据以上几种数据类型的读取方法，加入for循环批量读取数据以提高效率，除了for循环，也可以尝试使用apply函数和map函数进行批量处理，但个人感觉for循环还是更容易理解。注意：每种数据读取方式在转化为批量读取的时候稍微存在一些不同，这里为了后续引用方便直接一一列出：

首先是路径的处理，我们需要获得完整的路径名用以数据读取，还需要单独的文件名用以样本命名，即构建对象时project的赋值。路径获取通过list.files()函数实现。

首先获取文件夹所在路径，base_name为每个包含测序数据的文件夹的名称，full_name是包含前缀的整个路径。

xxxxxxxxxx
full_name = list.files("D:/R/test_folders",full.names = TRUE)
base_name = basename(full_name)

10X数据批量读取：

xxxxxxxxxx
scRNAlist <- list()
for (i in seq_along(full_name)) {
counts <- Read10X(data.dir = full_name[i]) #读取稀疏矩阵
scRNAlist[[i]] <- CreateSeuratObject(counts = counts,project = base_name[i],min.features = 200, min.cells = 3)}

h5数据批量读取：

xxxxxxxxxx
scRNAlist <- list()
for (i in seq_along(full_name)) {
counts <- Read10X_h5(file = full_name[i])
scRNAlist[[i]] <- CreateSeuratObject(counts = counts,project = base_name[i],min.features = 200,  min.cells = 3)}

txt.gz或csv.gz格式数据批量读取：

xxxxxxxxxx
for (i in seq_along(full_name)) {
  counts <-  read.table(gzfile(full_name[i]),header = T,row.names = NULL)
  counts <- counts[!duplicated(counts$row.names),]
  counts <- na.omit(counts)
  rownames(counts) <- counts$row.names
  counts <-counts[,-1]
  scRNAlist[[i]] <- CreateSeuratObject(counts,project = base_name[i],min.features = 200,min.cells = 3)
}

RData格式批量读取

单个的RData格式直接读取就行，然而多个RData格式读取后按照常规方法反而不好处理，因为读取进来的数据是多个不同的构建好的Seurat对象，并且名称是已经由数据上传的作者进行的命名，我们需要像前面的批量处理方法一样，将所有的对象按照规律放入一个列表中进行批量处理。下面的代码中，mget()函数用于在环境中获取与字符串向量base_name中元素同名的变量，并存放于列表scRNAlist中进行下一步处理。

xxxxxxxxxx
dir_name <- list.files("/mnt/e/R/R_work_directory/16samples/",full.names = TRUE)
base_name = basename(dir_name) %>%  substr(1,4) %>% paste0("scRNA_",.)
for (i in seq_along(dir_name)){load(dir_name[i])} 
scRNAlist <- mget(base_name)

构建完成的scRNAlist实际上是一个由Seurat对象组成的列表，每个对象scRNAlist[[i]]构成列表的一个元素，接下来的质控流程与单个样本的标准流程相似，但需要注意需要针对列表的操作，因此，在merge前，下面的质控过程同样需要用到for循环对列表内的每一个Seurat对象进行批量操作。

质量控制QC

质控的指标主要是"nFeature_RNA","nCount_RNA","mt_percent"，红细胞基因"HB_percent"在肿瘤微环境中似乎意义不大，为简化代码这里省去"HB_percent"指标。

批量计算线粒体和红细胞比例

xxxxxxxxxx
for (i in seq_along(scRNAlist)) {
  sc <- scRNAlist[[i]]  #获取scRNAlist中的第i个Seurat对象。
  sc[["mt_percent"]] <- PercentageFeatureSet(sc,pattern = "^MT-") #计算以"^MT-"开头的基因比例，即线粒体基因的比例。
  scRNAlist[[i]] <- sc   #将sc赋值给scRNAlist[[i]]
  rm(sc)  #删除sc
}

细胞亚群重聚类（以髓系重聚类为例）

对单细胞数据进行上述Seurat标准流程后，就可以开始正式按照我们的实验设计或需要进行个性化分析了，第一个步骤是要对研究的细胞类型进行进一步的重新降维聚类和亚群的鉴别，如研究肿瘤细胞，或者T细胞，或者髓系细胞，这里以髓系细胞为例。

重聚类的过程和前面的PCA降维聚类过程一直到细胞注释的流程是完全一样的，其本质就是将我们感兴趣的细胞类型从整个大的Seurat对象中以子集的形式取出来，只分析这部分细胞。
